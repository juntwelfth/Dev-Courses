Search Engine Optimization = 메타 태그를 이용한 검색엔진 최적화

웹페이지를 작성할 때 검색에 잘 노출되도록 HTML의 meta tag를 이용한 검색엔진 최적화기술
메타태그는 HTML 페이지의 head 섹션에 추가됨


<title>검색에서 노출되는 타이틀 부분</title>
    검색엔진의 제목부분에 표시될 수 있는 아주 중요한 정보
    검색결과에 반영되며 title 태그의 콘텐츠는 주로 결과의 첫 번째 행에 나타남
    타이틀은 페이지의 콘텐츠를 정확하게 설명하는 제목이어야 함
    웹 페이지마다 고유한 title 태그를 작성해줘서 웹페이지의 분별력을 높여줘야함
    콘텐츠를 정확하게 설명해야 하고, 간결하면서 내용을 포함해야 함
    1. 간결하면서 내용을 포함해야 한다
    2. 콘텐츠를 정확하게 설명해야 한다
    3. 웹페이지의 특성을 나타내야 한다


<meta name="description" content="검색에서 노출되는 설명부분">
    검색시 짧은 단락과 1~2개의 문장으로 정보가 표시되는 부분
    1-2개의 문장이나 짧은 단락을 사용하는 것이 좋다


<meta name="keywords" content="검색에서 노출되는 키워드부분">
    그 문서에 해당하는 키워드를 작성한다


URL
    URL의 주소를 좀 더 알아보기 좋게 만들기
    웹페이지의 주소에 컨텐츠의 내용을 포함하도록 하기 ex) 뉴스 사이트라면 google.com/news
    그 컨텐츠의 내용을 잘 표현하는 것으로 작성하며 과도하게 키워드 반복은 피하기 ex) google.com/news-todaynews-news

    하나의 페이지에 대해 여러 URL이 존재할 경우 대표 URL(Canonical URL)을 사용하는 것이 좋음 
    나머지 URL에 대해서는 대표 URL로 301 리다이렉션 시키거나 페이지 내에 rel=“canonical” 표시를 추가해 주는 것이 좋음
    특히 메인 페이지인 경우 하나의 대표 URL만 갖도록 나머지 URL에 대해서는 301 리다이렉션 처리를 해 줄 것을 권장합
    ex) www.example.com 과 example.com 둘 다 가능한 경우, 둘 중 하나를 대표 URL로 정하고 나머지는 리다이렉션 처리


Link
    링크를 통해서 다른 페이지로 이동하게 만들기
    검색엔진에 노출을 많이 시키려면 웹페이지 뿐만이 아니라 링크를 많이 포함하면 좋음
    웹페이지 제작시 사이트의 이동은 하이퍼텍스트를 이용
    검색엔진에 노출을 많이 시키려면 링크를 많이 포함하기
    주의해야 할 점은 검색 엔진을 위한 것이 아닌 사용자를 위한 콘텐츠를 작성해야 한다는 것

    사용자의 편의를 고려하지 않는 검색 엔진만을 위한 사이트는 오히려 검색 엔진으로부터 웹스팸으로 인정되어 차단될 수 있음


<h1...h6>
    한 페이지 내에 너무 많은 제목 태그들이 사용되는 것은 좋지 못함


유용한 오류 페이지 반환
    웹사이트에서 없는 URL을 호출하거나 기타 여러 이유로 존재하지 않는 페이지를 호출할 경우 흔히 ‘404 페이지’를 반환하는데, 이 때 404 페이지는 “페이지가 없습니다”라는 내용만 덜렁 출력하는 페이지 보단 사이트에 맞는 맞춤형 오류 페이지를 보여주는 것이 좋음

    이 때 HTTP 상태 코드 역시 HTTP 규약에 따라 오류 내용에 맞는 상태 코드를 반환해야 함
    즉 404 페이지인 경우 HTTP 200 OK가 아닌 HTTP 404 Not Found를 반환하도록 구성
    구글 검색엔진의 경우 404 코드가 반환되어야 할 페이지가 200 OK로 표시될 경우, 'Soft 404 Error'라고 하는 오류로 처리하기도 하니 주의를 요함


<a>...</a>
    앵커 태그(<a> 태그)는 페이지 간 이동을 위한 태그임
    앵커 태그를 작성할 때는 이동할 페이지의 내용을 함축하는 간결한 텍스트를 사용해야 하며, 글자로 쓸 내용을 이미지로 만드는 것은 좋지 못함


rel="nofollow"와 robots 메타 태그
    앵커 태그에서 검색엔진이 따라가기를 원치 않는 경우에는 다음과 같이 rel=“nofollow” 속성을 추가할 수 있음

    <a href="https://blog.example.com/spam-page" rel="nofollow">스팸 페이지</a>

    또한 페이지 단위로 검색엔진의 접근을 막기 위해 HTML <head> 영역 내에 다음과 같이 robots 메타 태그를 추가할 수도 있음

    <meta name="robots" content="noindex">

    이 때 content 값에 noindex를 사용하면 해당 페이지는 검색 결과에서 제외되며, nofollow를 사용하면 검색로봇이 해당 페이지 내의 링크를 따라가지 않도록 설정할 수 있음 기본 설정은 index, follow.


Mobile
    구글은 사이트가 모바일 친화적인지 검사할 수 있는 모바일 친화성 테스트 도구를 제공함

    반응형 웹 디자인
        웹 브라우저가 웹문서의 가로폭을 기기의 스크린 크기에 맞게 자동적으로 조절하는 기법 
        사용자 기기와 상관없이 동일한 URL에 동일한 HTML 코드를 게재하지만 화면 크기에 따라 다르게 렌더링 할 수 있음
        반응형 디자인은 Google에서 권장하는 디자인 패턴.

    동적 게재
        기기와 관계없이 동일한 URL을 사용하지만 서버가 사용자의 브라우저에 대해 알고 있는 정보를 기준으로 기기 유형에 따라 서로 다른 버전의 HTML을 생성하는 방식
        이러한 경우 동일한 URL이 모바일 검색과 데스크톱 검색에 보여 질 것이며, 결과물에 대하여 사용자는 각각 다른 버전에 맞는 콘텐츠를 보게됨
        이 때 주의해야 할 사항은, 이러한 설정을 잘못하게 되면 사이트는 “클로킹(cloaking)”이라는 Google의 가이드라인에 위배되는 행위를 한 것으로 취급되어 불이익을 받을 수도 있음
        일반적으로 클로킹은 사용자와 검색엔진에 각각 다른 콘텐츠를 제공하는 방식으로 검색 품질을 저해하는 행위를 의미합니다.

    별도 URL
        각 기기별로 URL을 구성하는 방식으로, 이 설정은 사용자의 기기를 감지한 다음 Vary HTTP 헤더와 함께 HTTP 리다이렉션을 사용하여 적절한 페이지로 리다이렉션 함
        이 때 두 버전을 운영하는데 있어서, 사용자를 위해 내용이 최대한 일치하도록 콘텐츠를 작성해야 함.(구글 가이드 상에는)
        ex) 쇼핑 사이트의 경우, 사용자가 모바일을 통해 데스크탑용 상품관련 URL을 접근하려고 할 때, 리다이렉션되는 모바일용 페이지가 모바일용 홈페이지가 아닌, 해당 상품의 모바일용 페이지가 되어야함
        또한 별도 모바일 페이지를 제공하는 경우에는 <link> 태그를 활용하여 데스크톱 사이트의 URL을 명시적으로 지정하는 것이 좋음
        특히 이 작업은 네이버의 검색로봇이 데스크톱 및 모바일 사이트에 방문할 때 콘텐츠가 중복으로 처리되지 않도록 알려주는 역할을 한다
        네이버 검색엔진 최적화 가이드에는 예를 들어 여러분이 데스크톱 사이트로 http://www.mysite.com이 있고 그와 대응하는 모바일 사이트인 http://m.mysite.com이 있다고 가정한다면 http://m.mysite.com 내의 페이지에 아래와 같이 데스크톱의 URL을 명시하는 것이 좋다고 나와있음

        모바일 사이트의 메인 페이지 ex) http://m.mysite.com/
            <head>
            <link rel="canonical" href="http://www.mysite.com/">
            </head>

        모바일 사이트의 개별 웹 페이지 ex) http://m.mysite.com/article/article1.html
            <head>
            <link rel="canonical" href="http://www.mysite.com/article/article1.html">
